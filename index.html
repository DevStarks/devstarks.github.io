Slakk

A group-messaging app inspired by Slack

Ruby on Rails, JavaScript, React / Redux, PostgreSQL

Live Github

Slakk allows users to communicate with each other in an organized way. Users can create and join public channels that are centered around specific topics. Users can also have private conversations using the direct-message or group-message features.

Real-time message updating was implemented using web-socket technology through a third-party API, Pusher. By leveraging ActiveRecord to preload necessary data before making SQL queries, data retrieval from the back-end is efficient and easily scalable. Slakk features a custom front and back end authentication that safely persists passwords to the database as hashes using the BCrypt hashing algorithm. By utilizing single-page architecture and fetching data using AJAX requests exclusively, Slakk requires 0 page reloads from login to logout, making navigation quick and seamless.

Orbital

A solar-sytem inspired sequencer

JavaScript, jQuery, Easel.JS, Tone.js

Live Github

Orbital is a solar-system inspired musical instrument for the browser. It works by looping 2, 4, or 8 measures and placing new notes, represented by planets, in orbit as the user inputs them. Each tone has its own orbital path and triggers a synth sound when it reaches its point of origin.

Orbital is built on an algorithm that determines an element's rate of motion based on user-determined tempo and number of measures. The EaselJS library was used to create canvas elements that represent planets that trigger synthesizers based on their coordinates. Multiple jQuery event-listeners are employed to give the user dynamic control over the display.

stORM

An object relational mapping library

Ruby

Github

stORM is an Object-Relational Mapping library inspired by ActiveRecord. stORM provides a base class `SQLObject` that, when inherited from, allows Ruby classes to easily query their related database table and return Ruby objects representing that data. These database-connected subclasses can be mapped to each other through associations.

stORM employs metaprogramming to dynamically create instance-methods based on object attributesand supports the mapping of single and multi-step associations that cross-reference database tables. By infering table-names, primary-keys, and foreign-keys under-the-hood, associations can be defined in an elegant way.
